面经（2024.11.25-2024.12.1）

SpringCloud

  • SpringCloud 是分布式微服务架构的一站式解决方案，是多种微服务架构落地技术的集合体，也被称为微服务全家桶；

核心组件与功能：

  • 服务注册与发现
  • 配置中心
  • 负载均衡
  • 服务熔断
  • 服务调用

‍

Redis 大 Key

  • 大 Key 指的是存储在 Redis 中的键值对，其中键对应的 value 占用了较大的内存空间，或包含了大量的元素

‍

常见场景

  • 缓存大数据
  • 明星或网红粉丝列表
  • 商品页所有信息

‍

排查大 Key

BIGKEYS

Redis 自带的 BIGKEYS 命令可以查询当前 Redis 中所有的 key 信息，对整个数据库中的键值对大小情况进行统计分析。

它会查出每种数据结构的最大 Key，但不能根据某个容量进行筛选。

‍

MEMORY USAGE

Redis 4.0+ 后推出了 MEMORY USAGE 命令，该命令可以返回指定 key 的内存使用情况，返回使用的内存的字节数。

‍

OBJECT

  • OBJECT encoding 可以查看键值对象的编码类型，不同的编码类型可能暗示了键值的复杂程度和大小。

例如：如果一个字符串类型的键采用了 raw 编码且长度很长，那么它可能是一个大 Key

‍

解决大 Key

  1. 拆分大 Key

  2. 使用压缩算法

  3. 使用合适的数据结构和存储方式

  4. 设置合理的过期时间

    1. 若大 Key 中的数据不是一直被需要的，可以通过设置过期时间让 Redis 在一定时间后自动删除，可以避免大 Key 长期占用内存，导致内存泄露。

  5. 加强监控和管理

    1. 建立对 Redis 的监控系统，实时监测大 Key 的出现和内存使用情况。当发现大 Key 或内存占用过高时，及时发出预警（Redis Insights、Prometheus）

‍

大 Key 删除

删除大 Key 时注意：使用 UNLINK 命令代替 DEL 命令来删除大 Key——UNLINK 命令会立即返回，后台异步删除数据，避免阻塞。

‍

小结

Redis 大 Key 问题会让 Redis 服务阻塞，无法响应其它命令，可能会导致客户端响应超时等问题。

排查大 Key 问题可以使用 BIGKEYS、MEMORY USAGE、OBJECT 等命令；

解决方案有：拆分大 Key、压缩数据、使用合适数据结构和存储方式、设置合理过期时间、以及加强监控和管理手段

HashMap 的遍历方式

遍历 key 和 value

  • 使用 map.entrySet() 最常用 最优解
  • 使用迭代器 最不常用
  • 使用 Stream API（Java 8） 适合大规模数据处理
  • 使用 forEach （Java8）最简洁

遍历 key

  • 使用 map.keySet() ，也可以获取 value 效率较低 不推荐

遍历 value

  • 使用 map.values() 只需要遍历值使用

String、StringBuffer、StringBuild 的区别

  1. 可变性

  • String 不可变，底层由 final 修饰，每次被创建出来，其实是在线程池中创建了一个对象，然后将指针引用过去。

    • 所谓修改 String 字符串实际上是在线程池创建了一个新的字符串对象，然后将引用指向新对象。

  • StringBuffer 和 StringBuilder 是可变的，可以通过其提供的 append 方法去对字符串进行修改

  1. 线程安全性
  • String 由于底层是不可变的，因此是线程安全的
  • StringBuffer 由于其公开方法都是由 synchronzied 修饰的，意味着同一时刻只能有一个线程访问 StringBuffer 对象，因此是线程安全的
  • StringBuilder 由于其方法没有被 synchronzied 修饰，线程不安全，多线程使用需要注意
  1. 性能
  • String 由于每次修改都要创建新对象，因此在频繁修改字符串场景下性能较低
  • StringBuffer 由于底层方法被 synchronzied 修饰，所以性能会比 StringBuilder 较低
  • StringBuilder 单线程情况下，由于没有锁的开销，所以性能较高
  1. 使用场景

  • String——适合字符串内容不经常修改

  • StringBuilder——适合单线程，且对字符串大量修改的场景

  • StringBuffer——适合多线程，且对字符串大量修改的场景
